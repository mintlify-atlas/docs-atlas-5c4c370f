<Note>
  **Three steps to production:** Set up shadcn/ui, initialize Convex, add a component. You'll have a working app in minutes.
</Note>

## Prerequisites

Before you begin, make sure you have:

- **Node.js 18+** installed on your machine
- A **React project** (Next.js, Vite, TanStack Start, or React Router)
- Basic familiarity with React and TypeScript

<Tip>
  Starting from scratch? The shadcn/ui setup will create a new project for you.
</Tip>

## Step 1: Set up shadcn/ui

Convex UI components are built on shadcn/ui. Choose your framework and follow the official setup guide:

<CardGroup cols={3}>
  <Card title="Next.js" icon="react" href="https://ui.shadcn.com/docs/installation/next">
    Recommended for full-stack apps with SSR
  </Card>
  <Card title="Vite" icon="bolt" href="https://ui.shadcn.com/docs/installation/vite">
    Fast development for SPAs and client-side apps
  </Card>
  <Card title="TanStack Start" icon="layer-group" href="https://ui.shadcn.com/docs/installation/tanstack">
    Modern routing with React Router foundation
  </Card>
</CardGroup>

<Info>
  **Already using shadcn/ui?** Skip to Step 2. Convex UI works with your existing setup.
</Info>

The shadcn/ui setup will:
- Configure Tailwind CSS
- Set up your `components.json` config file
- Install base UI dependencies
- Create your component directory structure

## Step 2: Initialize Convex

Install Convex and initialize your project:

<CodeGroup>
```bash npm
npm install convex @convex-dev/auth
npx convex dev
```

```bash pnpm
pnpm add convex @convex-dev/auth
pnpm convex dev
```

```bash yarn
yarn add convex @convex-dev/auth
yarn convex dev
```
</CodeGroup>

<Info>
  **No account required** for local development. Convex runs entirely on your machine until you're ready to deploy.
</Info>

The `convex dev` command will:

<Steps>
  <Step title="Create the convex directory">
    Initializes `convex/` with starter files and schema
  </Step>
  <Step title="Generate environment variables">
    Adds `NEXT_PUBLIC_CONVEX_URL` (or `VITE_CONVEX_URL`) to your `.env.local` file
  </Step>
  <Step title="Start the dev server">
    Launches a local Convex backend with hot reload
  </Step>
  <Step title="Deploy functions automatically">
    Watches for changes and redeploys instantly
  </Step>
</Steps>

### What gets created

After initialization, your project will have:

```text
your-project/
├── convex/
│   ├── _generated/         # Auto-generated TypeScript types
│   └── schema.ts           # Database schema (initially empty)
├── .env.local              # Convex deployment URL
└── convex.json             # Convex configuration
```

<Tip>
  Keep `npx convex dev` running while you develop. It provides hot reload for backend functions.
</Tip>

Now you can add any Convex UI component. Let's start with realtime chat:

  <Tab title="Next.js">

  </Tab>

    ```bash
    npx shadcn@latest add "https://convex-ui.vercel.app/r/realtime-chat-react"
    ```
  </Tab>
  <Tab title="TanStack Start">

This installs:

<CardGroup cols={2}>
  <Card title="Frontend" icon="browser">

## Step 4: Use the component

Import and use the chat component in your app:

<Tabs>
  <Tab title="Next.js">
    ```tsx app/page.tsx
    import { RealtimeChat } from "@/components/realtime-chat";
    
    export default function Home() {
      return (
        <main className="container mx-auto p-8">
          <RealtimeChat 
            roomName="general" 
            username="Alice" 
          />
        </main>
      );
    }
    ```
  </Tab>
  <Tab title="React (Vite)">
    ```tsx src/App.tsx
    import { RealtimeChat } from "@/components/realtime-chat";
    
    function App() {
      return (
        <div className="container mx-auto p-8">
          <RealtimeChat

    ```
  </Tab>

    ```tsx routes/index.tsx

    ```

</Tabs>

### Configure email provider

For password reset emails, configure Resend in the Convex dashboard:

<Steps>
  <Step title="Get a Resend API key">
    Sign up at [resend.com](https://resend.com) and create an API key
  </Step>
  <Step title="Add to Convex dashboard">
    Go to your project settings → Environment Variables
  </Step>
  <Step title="Set required variables">
    ```bash
    AUTH_RESEND_KEY=re_...
    AUTH_EMAIL_FROM=noreply@yourdomain.com
    ```
  </Step>
</Steps>

<Warning>
  OAuth providers (GitHub, Google) require additional setup. See the authentication docs for callback URL configuration.
</Warning>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Components not installing correctly">
    Try deleting your `components.json` file and running the shadcn/ui init command again for your framework. This resets your configuration to the latest defaults.

    ```bash
    rm components.json
    npx shadcn@latest init
    ```
  </Accordion>

  <Accordion title="Convex dev server not starting">
    Make sure you have Node.js 18+ installed:

    ```bash
    node --version
    ```

    If you're behind a firewall, you may need to configure proxy settings. Check the [Convex troubleshooting guide](https://docs.convex.dev/production/hosting/hosting-with-a-reverse-proxy).
  </Accordion>

  <Accordion title="TypeScript errors in generated files">
    Restart your TypeScript server in VS Code:

    - Press `Cmd/Ctrl + Shift + P`
    - Type "TypeScript: Restart TS Server"
    - Press Enter

    The `_generated` folder updates automatically when `convex dev` is running.
  </Accordion>

  <Accordion title="Environment variables not working">
    Make sure your `.env.local` file has the correct variable names:

    **Next.js:**
    ```bash
    NEXT_PUBLIC_CONVEX_URL=https://...
    CONVEX_DEPLOYMENT=dev:...
    ```

    **Vite, TanStack, React Router:**
    ```bash
    VITE_CONVEX_URL=https://...
    CONVEX_DEPLOYMENT=dev:...
    ```

    Restart your dev server after changing environment variables.
  </Accordion>
</AccordionGroup>
